# Phase 2: GitHub Client

## Goal

Implement the HTTP client that queries GitHub Releases API and downloads assets.

## Tasks

### 2.1 Create interface and types

**File:** `internal/update/github.go`

```go
type Release struct {
    Tag    string
    Assets []Asset
}

type Asset struct {
    Name string
    URL  string
}

type ReleaseChecker interface {
    LatestRelease(ctx context.Context) (*Release, error)
    DownloadAsset(ctx context.Context, url string, dest string) error
}
```

### 2.2 Implement `GitHubClient`

**File:** `internal/update/github.go`

```go
type GitHubClient struct {
    httpClient *http.Client
    repo       string // "svetozarm/scry"
}

func NewGitHubClient() *GitHubClient
```

`LatestRelease()`:
- `GET https://api.github.com/repos/svetozarm/scry/releases/latest`
- Header: `Accept: application/vnd.github+json`
- No auth header (REQ-NF-006)
- Timeout: 10s via context
- Parse JSON: extract `tag_name` and `assets[].{name, browser_download_url}`
- Map HTTP errors to `ErrUpdateAPI`

### 2.3 Implement `DownloadAsset()`

**File:** `internal/update/github.go` (same struct)

Strategy:
1. Create temp file in destination directory
2. HTTP GET with context
3. Stream body to temp file via `io.Copy`
4. On success: close file, rename to `dest`
5. On failure: close file, `os.Remove`, return error

Key detail: the `dest` path is provided by the caller (orchestrator), which ensures it's in the same directory as the binary.

### 2.4 Write tests

**File:** `internal/update/github_test.go`

Test cases using `httptest.Server`:
- **Success**: return valid JSON with tag and assets → verify parsed Release
- **Non-200**: return 503 → verify `ErrUpdateAPI`
- **Rate limited**: return 403 → verify `ErrUpdateAPI` with rate limit message
- **Malformed JSON**: return garbage → verify error
- **Timeout**: use cancelled context → verify `ErrUpdateAPI`
- **Download success**: serve file content → verify dest file matches
- **Download interrupted**: cancel context mid-transfer → verify temp file cleaned up

## Acceptance Criteria

- `LatestRelease()` correctly parses a real GitHub API response shape
- All HTTP error codes map to `ErrUpdateAPI`
- Downloaded files match served content byte-for-byte
- Partial downloads leave no temp files behind
