# Phase 3: Checksum Verification

## Goal

Implement SHA-256 checksum verification against GoReleaser's `checksums.txt` format.

## Tasks

### 3.1 Create `VerifyChecksum()` function

**File:** `internal/update/checksum.go`

```go
func VerifyChecksum(archivePath, archiveName, checksumsPath string) error
```

### 3.2 Implement checksums.txt parsing

GoReleaser format: `<sha256hex>  <filename>` (two spaces between hash and name).

```go
func parseChecksums(checksumsPath string) (map[string]string, error)
```

Returns a map of `filename → expected_hash`. If the file can't be read or parsed, wrap as `ErrChecksumMismatch`.

### 3.3 Implement SHA-256 computation

```go
func computeSHA256(filePath string) (string, error)
```

- Open file, create `sha256.New()`, `io.Copy` into hasher.
- Return lowercase hex string.
- Compare case-insensitively with expected value from checksums.txt.

### 3.4 Write tests

**File:** `internal/update/checksum_test.go`

| Test Case | Setup | Expected |
|---|---|---|
| Match | Create file, compute real hash, write to checksums.txt | nil error |
| Mismatch | Write wrong hash in checksums.txt | `ErrChecksumMismatch` |
| Missing entry | checksums.txt exists but doesn't contain the filename | `ErrChecksumMismatch` |
| Malformed file | checksums.txt with garbage content | `ErrChecksumMismatch` |

## Acceptance Criteria

- Correct SHA-256 passes verification
- Any mismatch returns `ErrChecksumMismatch`
- Missing entries are treated as mismatches (fail-safe)
