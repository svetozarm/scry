# Phase 7: Verification & Cleanup

## Goal

Ensure the feature works end-to-end across all platforms and update documentation.

## Tasks

### 7.1 Run full test suite

```bash
go test ./... -v
```

All existing tests must still pass. New tests in `internal/update/` and `cmd/update_test.go` must pass.

### 7.2 Manual end-to-end test

1. Build with a version: `go build -ldflags="-X main.version=v0.0.1" -o scry .`
2. Create a GitHub release with a tag newer than v0.0.1 (or use an existing one).
3. Run `./scry update` and verify it downloads, verifies, and replaces.
4. Run `./scry update` again and verify "already up to date".
5. Build without version: `go build -o scry .`
6. Run `./scry update` and verify dev build warning.

### 7.3 Cross-compilation check

Verify the build tags compile correctly for all platforms:

```bash
GOOS=linux GOARCH=amd64 go build ./...
GOOS=linux GOARCH=arm64 go build ./...
GOOS=darwin GOARCH=amd64 go build ./...
GOOS=darwin GOARCH=arm64 go build ./...
GOOS=windows GOARCH=amd64 go build ./...
GOOS=windows GOARCH=arm64 go build ./...
```

Ensures `replace.go` and `replace_windows.go` build tags are correct.

### 7.4 Update README

Add a section documenting `scry update`:
- What it does
- Example output
- Note about permissions (may need `sudo` on Unix if installed to `/usr/local/bin`)
- Note about dev builds

## Acceptance Criteria

- `go test ./...` passes
- Cross-compilation succeeds for all 6 targets
- Manual test confirms full update flow
- README documents the new command
